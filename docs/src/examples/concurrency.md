# Concurrency

[`examples/concurrent_primes.ryx`](https://github.com/ryx-lang/ryx/blob/main/examples/concurrent_primes.ryx) shows how Ryx uses fibers and channels for lightweight concurrency.

## Run It

```bash
./ryx run examples/concurrent_primes.ryx
```

Observed output:

```text
26
```

## Key Ideas

The example:

- Spawns four workers with `spawn`
- Uses a buffered `channel<Int>(100)` for communication
- Splits a numeric range across workers
- Sends `-1` as a completion sentinel per worker

## Relevant Snippet

```ryx
let results = channel<Int>(100);

spawn { check_range(2, 27, results) };
spawn { check_range(27, 52, results) };
spawn { check_range(52, 77, results) };
spawn { check_range(77, 102, results) };
```

## Related Commands

Inspect lowered forms and generated bytecode for a concurrent program:

```bash
./ryx run -dump-mir examples/concurrent_primes.ryx
./ryx run -dump-bytecode examples/concurrent_primes.ryx
```
