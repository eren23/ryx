# Task: Fix VM stack underflow panics in arithmetic operations

E2E tests fibonacci.ryx, quicksort.ryx, and matrix_multiply.ryx crash with Go-level panics: 'index out of range [-1]' in fiber.go:89 (Pop) and 'index out of range [1] with length 1'. These indicate the VM is trying to pop from an empty stack or access stack positions that don't exist. This means the bytecode being generated has mismatched push/pop counts for some operation sequences. Common causes: binary operations not finding both operands on stack, function calls not properly setting up arguments, or return values not being handled correctly. Debug by examining the bytecode generated for fibonacci.ryx (a simple case) and tracing the stack state. Fix the codegen to emit correct stack-balanced bytecode.

Target files: pkg/codegen/codegen.go, pkg/vm/vm.go, pkg/vm/fiber.go
