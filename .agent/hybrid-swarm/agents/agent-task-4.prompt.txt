# Task: Fix recursive ADT dispatch causing call depth exceeded

Multiple E2E tests (binary_tree, linked_list, pattern_matching, adt_expression, red_black_tree) fail with 'runtime error: call depth exceeded (max 1024)'. These all involve recursive functions over algebraic data types with pattern matching. The compiled pattern match decision trees or the recursive function calls are likely generating infinite recursion. The root cause is probably in how pattern matching on enum variants is compiled (pkg/hir/match_compile.go) or how the VM dispatches match results. Check the generated bytecode for a simple recursive ADT function to see if the match dispatch is correct - it may be re-matching the same value instead of the destructured inner value, causing infinite recursion. Fix the match compilation or codegen to properly handle recursive ADT traversal.

Target files: pkg/hir/match_compile.go, pkg/hir/lower.go, pkg/codegen/codegen.go
